# Administración de 'repos' con git y GitHub

# Repositorios gestionados desde la terminal con .git

Genero un nuevo repositorio git y empiezo a supervisar cambios en los archivos del programa:

git init
git add .
git status
git commit -m "Inicio"

# Subir el repositorio a GitHub

Desde la terminal, dentro del directorio del proyecto, ejecuto:

git add .
git commit -m "detalle de la actualización"
git push

# Repositorio en GitHub para proyectos de VuePress

La documentación oficial de VuePress detalla como subir un proyecto con un pequeño programa que se ejecuta desde la terminal. Es importante crear primero un archivo .gitignore como se explica más arriba, para evitar subir todos los módulos y otros archivos no imprescindibles.

  1. Primero creo un proyecto vacío en GitHub.
  2. Añado el elemento base al archivo de configuración del proyecto config.js: config.js :
    base: "/Vue-Docs/",

ATENCIÓN

Esto es necesario para que se pueda publicar en un repositorio en GitHub. También va a modificar las rutas de acceso a los documentos del proyecto.

  1. Luego creo un archivo en la raiz del proyecto con la configuración para la publicación en GitHub. deploy.sh :
#!/usr/bin/env sh

# abort on errors
set -e

# build
npm run docs:build

# navigate into the build output directory
cd docs/.vuepress/dist

# if you are deploying to a custom domain
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'


# if you are deploying to https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO> master:gh-pages
git push -f git@github.com:PlantAnyTree/Vue-Docs.git master:gh-pages

cd -
  1. Le doy permisos de ejecución:
chmod deploy.sh +x
  1. Por último ejecuto el código desde una terminal:
./deploy.sh

ATENCIÓN

Lo que se sube es el código ya formateado como html y no el código fuente que se renderiza para crear el sitio.
