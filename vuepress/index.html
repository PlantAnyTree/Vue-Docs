# VuePress

# yarn o npm

La página oficial de VuePress aconseja instalar la aplicación con yarn. Durante el aprendizaje creé dos proyectos con npm. Los proyectos se cuelgan con mucha facilidad y tengo que reiniciar el servidor cada vez que quiero visualizar nuevo contenido. Por este motivo he decidido probar con yarn.

Instalación de npm con NodeJS.
Página Oficial de Yarn

# Nuevo proyecto VuePress con Yarn

Según la página oficial de vuepress, la forma más sencilla de crear un nuevo proyecto es con el comando:

yarn create vuepress vuepress-docs

Luego entramos en el directorio creado para el proyecto e instalamos las dependencias:

cd vuepress-docs
yarn install

# VuePress con npm

El administrador de paquetes npm (node package manager) se instala por defecto al instalar NodeJS.

# Proyecto predeterminado con VuePress

En el caso de VuePress, se puede crear un nuevo proyecto con una estructura ya definida con:

npx create-vuepress <nombredelproyecto>

El programa de instalación preguntará si queremos crear un blog o una web de documentación de alguna aplicación... También pregunta otros datos del proyecto como nombre, descripción, repositorio...

# Crear un proyecto vacío con npm

Si no queremos que nos determine la estructura del proyecto podemos hacer una instalación manual:

mkdir doc
cd doc
npm init -y

Una vez inicializado el proyecto se le añade el paquete VuePress:

npm install -D vuepress

# Últimos pasos

# Definir script de arranque

  • Se debe editar el archivo packajes.json
  • Se añade lo siguiente:
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }

# Directorio de contenido

  • Debo crear un directorio 'docs' dentro del proyecto.
  • Para configurar el proyecto necesito otro directorio con el nombre .vuepress
  • Por último hace falta un directorio donde guardar el contenido: public
mkdir -p docs/.vuepress/public

# Arrancar el proyecto:

Con yarn:

yarn docs:dev

Con npm:

npm run docs:dev

# Configuración

La configuración de la página se determina en el archivo /docs/.vuepress/config.js.

# Publicar

El directorio por defecto donde se monta el proyecto es '/docs/.vuepress/dist/'. Hay que asegurarse de que existe o crearlo:

mkdir -p /docs/.vuepress/dist

No todos los paquetes para publicar proyectos utilizan este directorio, pero no está de más crearlo, por si acaso.

# Firebase

Para publicar el proyecto en el servidor de Google hay que asegurarse de que tenemos instalado firebase-tools. Si no es así:

npm install -g firebase-tools

En mi caso necesito permisos de administrador.

Luego debo crear dos archivos en la raiz del proyecto:

firebase.json :

{
 "hosting": {
   "public": "./docs/.vuepress/dist",
   "ignore": []
 }
}

.firebaserc :

{
 "projects": {
   "default": "<YOUR_FIREBASE_ID>"
 }
}

Ahora compilo el código

npm run docs:build

Crea los archivos del proyecto. Y por último los subo al servidor:

firebase deploy
