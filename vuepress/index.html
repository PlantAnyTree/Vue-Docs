# VuePress

# Crear proyecto npm

mkdir doc
cd doc
npm init -y

# Crear proyecto git

En el caso de este proyecto he creado un repositorio git como se indica en: .git.
Sin embargo he terminado publicando el código según como se detalla en la documentación oficial de VuePress y se explica más abajo en este documento.
Es importante, de todas formas, crear un archivo .gitignore, para evitar subir todos los módulos...

# Definir script de arranque

  • Se debe editar el archivo packajes.json
  • Se añade lo siguiente:
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }

# Directorio de contenido

  • Debo crear un directorio 'docs' dentro del proyecto.
  • Para configurar el proyecto necesito otro directorio con el nombre .vuepress
  • Por último hace falta un directorio donde guardar el contenido: public
mkdir -p docs/.vuepress/public

# Arrancar el proyecto:

npm run docs:dev

# Publicar

El directorio por defecto donde se monta el proyecto es '/docs/.vuepress/dist/'. Hay que asegurarse de que existe o crearlo:

mkdir -p /docs/.vuepress/dist

No todos los paquetes para publicar proyectos utilizan este directorio, pero no está de más crearlo, por si acaso.

# GitHub

  1. Primero creo un proyecto vacío en GitHub (web).
  2. Tengo que añadir el elemento base al archivo de configuración del proyecto: config.js :
    base: "/Vue-Docs/",

ATENCIÓN

Esto va a hacer que se pueda publicar en un repositorio de GitHub, pero también va a provocar que se instale en un subdirectorio del servidor donde se suba y añade esa ruta a todos los enlaces del proyecto.

  1. Luego creo un archivo en la raiz del proyecto con la configuración para la publicación en GitHub. deploy.sh :
#!/usr/bin/env sh

# abort on errors
set -e

# build
npm run docs:build

# navigate into the build output directory
cd docs/.vuepress/dist

# if you are deploying to a custom domain
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'


# if you are deploying to https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO> master:gh-pages
git push -f git@github.com:PlantAnyTree/Vue-Docs.git master:gh-pages

cd -
  1. Le doy permisos de ejecución:
chmod deploy.sh +x
  1. Por último ejecuto el código desde una terminal:
./deploy.sh

ATENCIÓN

Lo que se sube es el código ya formateado como html y no el código fuente que se renderiza para crear el sitio.

# Firebase

Para publicar el proyecto en el servidor de Google hay que asegurarse de que tenemos instalado firebase-tools. Si no es así:

npm install -g firebase-tools

En mi caso necesito permisos de administrador.

Luego debo crear dos archivos en la raiz del proyecto:

firebase.json :

{
 "hosting": {
   "public": "./docs/.vuepress/dist",
   "ignore": []
 }
}

.firebaserc :

{
 "projects": {
   "default": "<YOUR_FIREBASE_ID>"
 }
}
